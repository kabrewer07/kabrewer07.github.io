SQL and NoSQL Databases

June 27th, 2014

How SQL databases and NoSQL databases model data is different in many ways. SQL databases have a relational approach and are more structured and strive to be efficient and reliable. NoSQL databases take more of a flexible, dynamic approach to structure in an effort to reduce limitations in storing data. Below are listed some specific differences between these two types of databases.

SQL (Relational)

  • Based on tables
  • Predefined schema
  • Vertically scalable (increase power of machines)
  • Uses SQL to find and manipulate the data
  • Database examples:
    • MySQL
    • Oracle
    • Sqlite
  • Best for hierarchial data storage

NoSQL (Distributed)

  • Based on documents, graphs databases, and a few other types
  • Dynamic, unstructured schema
  • Horizontally scalable (more machines)
  • Uses UnQL to collect documents and all have their own unique ways of accessing the data
  • Database examples:
    • MongoDB
    • CouchDb
    • Redis
  • Best for large data (such as documents)
