Rails!

July 3rd, 2014

Since we will be doing so much work with Rails over the course of DBC I decided that this would be a great topic to learn more about. Rails is NOT the same as Ruby in that it is the open sourced web application framework that runs using the Ruby development language. Rails strives to be a very efficient and more simple framework by reducing the need for repetitive code and using convention rather than configuration.

Technical Stuff

Ruby uses the Model-View-Controller framework. Luckily I have some experience with the Zend Framework, which is also MVC so I know a little bit about what each of these pieces does. The model interacts with database tables. The controller is made up of actions and handles the external requests and interactions with the models and views. The controller gives information to the view to be displayed and is basically what you see so it is usually made up of HTML.
