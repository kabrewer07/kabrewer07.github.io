Blocks and Procs and Lambdas Oh My!

June 22nd, 2014

Blocks

In Ruby a block is some code that can be run which can be combined with methods. The two ways it is run is by using do...end for multiple line blocks or curly brackets for single line blcoks. Below are some examples of how to use blocks.

numbers = [0, 1, 2, 3, 4]
numbers.each do |x|
  p x * 2
end
# => Prints 0, 2, 4, 6, 8 on separate lines
numbers.each { |x| p x }
# => Prints 0, 1, 2, 3, 4 on separate lines

In this example the block of code is combined with the .each method and is being run on the numbers array. In the first one we are multiplying each value in the array by two and outputting it. In the second we are just outputting the original value.

Procs

Blocks are not objects and can't be saved as variables so we have to repeat the code if we want to do the same thing again. Procs are objects and allow us to write the code out once and use it repeatedly by saving it to a variable. In code using a proc we use an "&" to change the proc into a block since the method is expecting a block. A proc will return the call immediately and does not continue with the method.

numbers = [0, 1, 2, 3, 4]
new_array = Proc.new { |x| x * 2 }
numbers.map!(&new_array)
# => [0, 2, 4, 6, 8]

# Another example

def greeting_call
	greeting = Proc.new { return "Hello!" }
	greeting.call
	p "Hello again!"
end
p greeting_call # => "Hello!"

Lambdas

Lambdas are very similar to procs in that they are also objects and also executes some code. They are different in that lambdas check the number of parameters passed in. Procs do not and give any missing arguments a value of nil. Another difference is that lambdas will give control back to the method once the block of code is run.

greeting = lambda { |x| x * 2 }
p greeting.call("hello") # => "hellohello"

# Another example

def greeting_call
	greeting = lambda { return "Hello!" }
	greeting.call
	p "Hello again!"
end
p greeting_call 
# => "Hello again! nil" on separate lines
