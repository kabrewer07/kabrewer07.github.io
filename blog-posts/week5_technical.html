Initialize Method and Instance Variables

June 15th, 2014

Initialize Method

When calling a Ruby class on an object, it is possible to pass in parameters/arguments when creating a new instance of the class. You do this by calling the new method with the parameters passed through it. To access these parameters from the class, you would first define a method called "initialize". Initialize is run when the "new" method is called on the class. Within this method you can use the parameters to create instance variables.

class Restaurant
	def initialize (name, location)
		@name = name
		@location = location
	end
end

Instance Variables

Usually the reason for creating an initialize method is so that we can pass in parameters to it when calling the "new" method. In order to use these parameters throughout the class in other methods, we need to set the parameters to instance variables. You can see in the example above where we created the name and location instance variables in the initialize method. Instance variables are accessible to all methods within that instance. These variables are different from local variables in that they have an "@" in front of the variable name. Local variables are only available in the method in which they are created. Instance variables are great for passing and manipulating information throughout the methods in a class.

Here is an example of how instance variables can be used in a class. Notice how we do not need to pass any parameters into the ad method!

class Restaurant
	def initialize (name, location)
		@name = name
		@location = location
	end
	def ad
		print "Check out #{@name}" at #{@location}!""
	end
end
